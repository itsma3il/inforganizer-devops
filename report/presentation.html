Contexte et Problematique

Defis actuels

  • Erreurs humaines lors des configurations manuelles
  • Temps de deploiement longs (heures/jours)
  • Absence de tracabilite des modifications
  • Rollback complexe et risque

Question de recherche

Comment concevoir une infrastructure permettant un deploiement automatise, reproductible et auditable ?

Objectifs du Projet

Priorite Objectif Critere de succes
1 Automatisation complete Zero intervention manuelle
2 Reproductibilite Infrastructure recreable en < 30 min
3 Securite Pas de credentials statiques
4 Haute disponibilite Zero downtime lors des deploiements
5 Tracabilite Audit trail complet via Git

Architecture GitOps

Paradigme GitOps : Git constitue la source de verite unique pour l'etat desire du systeme.

4 Principes fondamentaux

  • Declaratif : Description de l'etat desire, pas des etapes
  • Versionne : Historique complet dans Git
  • Tire automatiquement : Agent surveille et applique les changements
  • Reconciliation continue : Correction automatique des derives

Stack Technologique

AWS (VPC, EC2, ECR) Terraform Ansible Docker Kubernetes (k3s) GitHub Actions ArgoCD

Infrastructure as Code

  • Terraform : Provisionnement AWS
  • Ansible : Configuration k3s

Deploiement

  • GitHub Actions : CI/CD + OIDC
  • ArgoCD : GitOps operator

Pipeline CI/CD

Commit → Tests → Build → Push ECR → GitOps Update → ArgoCD Sync

Securite OIDC

  • Token genere a chaque execution
  • Expire apres 1 heure max
  • Lie au repository et branche
  • Zero credentials statiques

Avantages

  • Echec rapide (fail fast)
  • Tag SHA = immutabilite
  • [skip ci] evite les boucles
  • Self-healing ArgoCD

Deploiement Kubernetes

Haute Disponibilite

  • 2 replicas avec anti-affinite
  • RollingUpdate : maxUnavailable=0
  • Init container : attente PostgreSQL
  • Probes : Liveness + Readiness

Securite

  • Image Docker multi-stage
  • Utilisateur non-root
  • Secrets Kubernetes (non en clair)
  • Image finale : 49 Mo

Self-healing : Si un operateur modifie manuellement un deployment, ArgoCD restaure l'etat Git en < 3 secondes.

Discussion

Points Forts

  • Automatisation complete atteinte
  • Securite multicouche (OIDC, non-root)
  • Zero downtime (RollingUpdate)
  • Tracabilite via Git

Limites

  • SPOF : 1 seul master k3s
  • Absence de monitoring
  • Pas de rotation secrets
  • Backup DB non automatise

Ameliorations Proposees

Prometheus/Grafana Backup PostgreSQL k3s HA (3 masters) HashiCorp Vault

Conclusion

Ce projet demontre qu'une infrastructure DevOps de niveau professionnel est accessible dans un contexte academique.

Realisations cles
